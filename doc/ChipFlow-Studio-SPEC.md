# ChipFlow Studio — Product Specification (SPEC)

> **一句话定位**：面向嵌入式/电气工程师的 AI-Native 在线芯片平台，像 Claude Code 一样智能地辅助从方案分析到 PCB 出图的完整硬件开发流程。

---

## 1. 产品命名

| 候选名 | 含义 | 特点 |
|--------|------|------|
| **ChipFlow Studio** ✅ | Chip + Workflow，芯片设计工作流 | 突出"流程化"，读音响亮 |
| Emforge | Embedded + Forge（锻造） | 工业感强，暗示打造硬件 |
| CircuitMind | Circuit + Mind（智能） | 强调 AI 思考能力 |
| Cubex AI | CubeMX 的现代进化版 | 借势，但有商标风险 |
| AmpLab | Amplifier（放大器）+ Lab | 电气工程味，稍局限 |

**推荐：ChipFlow Studio** — 清晰传达"芯片+工作流+工作室"三层含义，注册域名可用 `chipflow.studio` 或 `chipflow.io`。

## 2. 技术栈

```
┌─────────────────────────────────────────────────────┐
│                   前端 (Frontend)                     │
│  Next.js 14 + TypeScript + TailwindCSS + Shadcn/ui   │
│  Monaco Editor (代码/配置编辑)                         │
│  ReactFlow (节点式画布，管脚映射/时钟树可视化)            │
│  Zustand (状态管理)                                    │
├─────────────────────────────────────────────────────┤
│                   后端 (Backend)                      │
│  Python FastAPI (主后端，科学计算生态好)                  │
│  Node.js Express (轻量网关 + WebSocket)                │
│  PostgreSQL + pgvector (RAG 向量检索)                  │
│  Redis (会话缓存 + 实时协作)                            │
│  MinIO/S3 (项目文件存储)                               │
├─────────────────────────────────────────────────────┤
│                 AI 层 (The Brain)                     │
│  Claude API (主模型，Agent 推理)                       │
│  LangChain/LangGraph (Skill 编排)                     │
│  RAG Pipeline (芯片数据手册/参考设计知识库)              │
│  pgvector (芯片参数语义搜索)                            │
├─────────────────────────────────────────────────────┤
│               基础设施 (Infra)                         │
│  Docker + Kubernetes                                   │
│  GitHub Actions (CI/CD)                               │
│  Clerk/Auth0 (用户认证)                                │
└─────────────────────────────────────────────────────┘
```

**为什么选 Next.js + Python FastAPI 双后端？**
- Next.js：SSR/SSG 优化首屏，Monaco Editor 生态好
- FastAPI：Python 科学计算库（numpy/scipy）直接复用，芯片参数搜索、电路计算天然优势

**布局设计理念：**
- 经典三栏 IDE 布局，工程师零学习成本（对标 VS Code / Claude Code 对话模式）
- 左侧项目树：文件式管理，符合工程习惯
- 中央画布：多 Tab 切换不同视图（原理图预览、PCB、BOM 表格、管脚映射图、时钟树）
- 右侧 AI 面板：对话 + Skill 快捷调用，像 Claude Code 的 `/skill` 机制
- 底部 Output 面板：日志/终端输出，AI 执行结果实时透出

## 4. UI 构想

### 4.1 设计语言

| 维度 | 选择 |
|------|------|
| 风格 | **Industrial Dark** — 深色背景 + 科技蓝 + PCB 绿点缀 |
| 字体 | JetBrains Mono (代码) + Inter (UI) |
| 图标 | Lucide Icons + 自定义芯片/PCB 图标 |
| 配色 | `#0a0e27` 深蓝黑底 + `#00d4ff` 科技蓝 + `#2ecc71` PCB绿 + `#f39c12` 警告橙 |
| 圆角 | 8px (卡片) / 6px (按钮) / 4px (输入框) |

### 4.2 关键页面与视觉风格描述

**登录/首页**
- 全屏深色渐变背景，中央大标题 "ChipFlow Studio" 带微妙的电路板线条动画
- 副标题："AI-Powered Chip Design Workflow"
- 一个输入框："描述你的项目需求…" — 像 ChatGPT 一样直接开始

**项目仪表盘 (Dashboard)**
- 卡片式项目列表，每个卡片显示芯片型号、最近修改时间、Skill 使用历史
- 顶部搜索栏 + "新建项目" CTA 按钮
- 左侧筛选：按芯片厂商（ST/TI/NXP/Microchip…）分类

**工作台 (Workspace)** — 核心页面
- 三栏布局见上方
- AI 面板支持 `/` 触发 Skill 菜单（浮层显示 20+ 个 Skill 图标+名称）
- Skill 执行时右侧面板流式输出思考过程（像 Claude Code 的 thinking）
- 中央画布支持暗色/亮色切换
- 管脚映射图使用 ReactFlow 节点图，芯片在中心，外设围绕
- PCB 预览使用 3D 渲染（Three.js）
- 所有操作可被 AI Agent 驱动

**Skill 市场 / 商店**
- 卡片网格展示 20+ 内置 Skill
- 每个 Skill：图标 + 名称 + 简短描述 + 使用次数
- 支持自定义 Skill 上传（社区生态）

## 5. 内置 Skill 体系（22 个）

```
┌─────────────────────────────────────────────────────────┐
│                 ChipFlow Skill System                    │
├───────────────┬───────────────┬─────────────────────────┤
│  方案与选型    │  硬件设计      │  生产与验证             │
├───────────────┼───────────────┼─────────────────────────┤
│ 1. 需求分析   │ 7. 原理图设计  │ 15. BOM 清单生成       │
│ 2. 芯片选型   │ 8. 管脚分配   │ 16. 成本估算            │
│ 3. 架构设计   │ 9. 时钟树配置  │ 17. 元件采购建议        │
│ 4. 方案对比   │ 10. 电源设计  │ 18. Gerber 审查         │
│ 5. 技术调研   │ 11. PCB 布局  │ 19. DFM 检查            │
│ 6. 风险分析   │ 12. 信号完整性 │ 20. 测试点规划           │
│              │ 13. EMC/EMI  │                       │
│              │ 14. 热分析    │                       │
├───────────────┴───────────────┴─────────────────────────┤
│                    固件与软件                             │
├───────────────┬───────────────┬─────────────────────────┤
│ 21. 外设驱动  │ 22. RTOS 配置 │ 23. Bootloader 设计     │
│ 代码生成      │               │                        │
└───────────────┴───────────────┴─────────────────────────┘
```

### Skill 详解（部分示例）

| # | Skill 名称 | 输入 | 输出 | AI 做了什么 |
|---|-----------|------|------|------------|
| 1 | **需求分析** `/req` | 自然语言描述（"做一个四轴无人机飞控"） | 结构化需求文档、关键参数表 | 对话式澄清需求→提取关键约束→生成规格书 |
| 2 | **芯片选型** `/chip-select` | 需求规格（GPIO数、频率、外设、成本） | Top 5 推荐芯片 + 对比表格 | RAG 搜索芯片数据库→多维评分排序→解释推荐理由 |
| 3 | **架构设计** `/arch` | 需求+选定芯片 | 系统框图、模块划分、接口定义 | 推理最优架构→生成 Mermaid 框图→检查冲突 |
| 7 | **原理图设计** `/schematic` | 架构+芯片+外设列表 | 原理图 Netlist + 设计说明 | 逐模块生成连接方案→参考设计库匹配→标注注意事项 |
| 8 | **管脚分配** `/pinmap` | 芯片型号 + 外设需求 | 管脚分配表 + 冲突报告 | 自动分配→检查电气冲突→备选方案→生成 CubeMX 可导入配置 |
| 9 | **时钟树配置** `/clock` | 芯片 + 频率需求 | 时钟树图 + PLL 配置参数 | 计算分频/倍频→检查精度误差→生成配置代码 |
| 11 | **PCB 布局** `/pcb` | 原理图 Netlist + 约束 | 布局建议 + 布线策略 | 分析信号流→分区建议→关键走线提示→生成设计规则 |
| 15 | **BOM 生成** `/bom` | 原理图 | BOM 表格 + 成本 + 货源 | 识别所有元件→匹配封装→查询实时价格→替代料推荐 |
| 17 | **元件采购** `/procure` | BOM 清单 | 各平台比价 + 采购链接 | 搜索 LCSC/Mouser/Digikey→比价→检查库存 |
| 18 | **Gerber 审查** `/gerber` | Gerber 文件 | 问题列表 + 修复建议 | 解析 Gerber→检查设计规则→对比行业标准 |
| 21 | **外设驱动生成** `/driver` | 芯片+外设配置 | HAL 层初始化代码 | 参考官方 SDK→生成驱动框架→添加注释 |

## 6. AI Agent 交互设计

### 核心体验：像 Claude Code 一样协作设计硬件

```
用户: "我要做一个智能灌溉控制器，太阳能供电，需要低功耗"
      "/chip-select"

Agent: 🔍 正在分析需求...
       ✅ 提取关键约束：低功耗、太阳能输入、GPIO ≥ 8、ADC ≥ 2
       📊 搜索芯片数据库...
       
       推荐 Top 3：
       1. STM32L431CBT6 — 超低功耗，12bit ADC，¥18.5
       2. MSP430FR5994 — FRAM，最低功耗，¥22.0
       3. nRF52840 — BLE + 低功耗，¥25.0
       
       我推荐 STM32L431，因为你还需要 ADC 采集土壤湿度，
       且 ST 的生态对新手友好。要我详细对比吗？

用户: "就 STM32L431 吧，帮我做管脚分配"

Agent: ✅ 已锁定芯片：STM32L431CBT6
       🦶 正在分配管脚...
       
       ⚠️ 检测到冲突：PA2 同时被 USART2_TX 和 ADC_IN2 占用
       💡 建议：USART2_TX 移到 PA9（默认 USART1_TX 所在组）
       
       管脚分配完成，已避让所有冲突。[/pinmap 输出见画布]
```

### Skill 触发方式

| 方式 | 操作 | 适用场景 |
|------|------|---------|
| `/skill名` | 在 AI 对话中输入 | 熟悉命令的老用户 |
| 点击 Skill 列表 | 右侧面板点击图标 | 浏览发现 |
| 自然语言 | 直接描述需求 | 新手，Agent 自动路由到对应 Skill |
| 右键菜单 | 在画布元素右键 | 上下文敏感操作 |

## 7. 开发路线图 (MVP 优先)

```
Phase 1: MVP (4-6 周)
├── 用户认证 + 项目 CRUD
├── AI 对话面板 (Claude API 接入)
├── 3 个核心 Skill: /chip-select, /pinmap, /schematic
├── 管脚映射可视化画布 (ReactFlow)
├── 芯片数据库 (STM32 + GD32 + ESP32 初始 200 款)
└── 项目导出 (PDF + Markdown)

Phase 2: 扩展 (6-10 周)
├── /pcb, /bom, /clock, /power 等 10 个新 Skill
├── PCB Gerber 在线预览 (Three.js 3D)
├── BOM 比价 (LCSC/Mouser API)
├── 外设驱动代码生成
├── 项目模板市场
└── 团队协作 (实时同步)

Phase 3: 生态 (长期)
├── 社区 Skill 市场 (自定义 Skill SDK)
├── KiCad/Altium/立创EDA 插件
├── CubeMX 配置导入/导出
├── 仿真集成 (SPICE 在线仿真)
├── 企业版 (私有部署 + SSO)
└── 硬件 CI/CD Pipeline
```

## 10. 风险与挑战

| 风险 | 等级 | 缓解措施 |
|------|------|---------|
| 芯片数据库质量 | 高 | 爬取+人工审核+社区贡献，初期聚焦 3-5 个主流品牌 |
| AI 输出准确性 | 高 | 输出必须标注置信度 + 建议人工复核 + 参考设计交叉验证 |
| 技术门槛 | 中 | Skill 模式降低门槛，自然语言即可驱动 |
| 电子工程师习惯 | 中 | 兼容 CubeMX 导入导出，降低迁移成本 |
| 成本 (API 调用) | 中 | 免费额度 + 本地模型辅助（小任务本地跑） |
